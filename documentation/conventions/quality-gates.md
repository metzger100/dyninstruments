# Quality Gates

**Status:** ✅ Implemented | Command graph and checker ownership map

## Overview

This convention maps the repository quality commands to their checker scripts and rule ownership. Use it to choose the
smallest local gate during iteration and the required final gate before finishing work. The local command graph is
authoritative. Tag pushes rerun the complete gate before GitHub publishes artifacts prepared and committed locally.

## Key Details

- `npm run check:all` is the default complete local gate.
- `.pre-commit-config.yaml` provides optional local fast-feedback hooks; it is not a replacement for the complete local
  gate.
- `check:core` runs `check:standard`, both typecheck scopes, package and contract tests, the focused-test configuration
  proof, smell/complexity/scaling policy checks, documentation checks, and the file-size gate in package-script order.
- `check:standard` owns the additive standard-tool layer: full-repository Prettier formatting (config/workflow files,
  all production/test/tool JavaScript, CSS, and Markdown), ESLint flat config, Stylelint, pinned actionlint workflow
  validation, and fail-on-detection jscpd duplication checking.
- `typecheck` owns both strict no-emit boundaries: production/config/tool declarations in `tsconfig.checkjs.json` and
  inventory-owned tests in `tsconfig.tests.json`.
- Prettier owns every maintained JS/MJS, CSS, and Markdown surface: `runtime/`, `cluster/`, `config/`, `shared/`,
  `widgets/`, `plugin.js`/`plugin.mjs`, `tests/`, `tools/`, `plugin.css` and widget/shared/test CSS, and root plus
  `documentation/**`, `.agents/skills/**`, and `exec-plans/active/**` Markdown. Completed execution plans and release
  notes are archival records and remain outside formatter ownership. `tests/contract/formatting-scope-contract.test.js`
  proves `format`/`format:check` cover every maintained file and that negative-fixture exclusions stay narrowly scoped
  by exact path.
- `check:smells` runs the static pattern smell gate; semantic smell contracts live in `test:contract`.
- `docs:check` starts with markdownlint-cli2, then runs Linkinator for local Markdown links/fragments and the retained
  project-specific graph, stale- phrase, shape, and reachability contracts.
- `test:coverage:check` owns Vitest coverage generation and native global/glob threshold enforcement.
- Required local gates use Node/Vitest only; no command may require Playwright, a downloaded browser, or a browser
  driver.
- Release creation is locally gated by `check:all` before package creation.
- `.github/workflows/publish-release.yml` reruns `setup` and `check:all` in a read-only tag quality job; the
  write-privileged publisher depends on it and never rebuilds release artifacts. The shared release-version tool
  validates the tag and tells the publisher whether to create a normal or prerelease GitHub Release.

## Delivery: pre-push hook, branch quality CI, and required ruleset

- `.githooks/pre-push` runs `npm run check:all`. `npm run hooks:install` sets `core.hooksPath=.githooks` and marks the
  hook executable; `npm run hooks:doctor` verifies both and prints the exact repair command when either drifts. CI does
  not depend on this local configuration.
- `.github/workflows/quality.yml` runs on pull requests targeting `main` and on pushes to `main`. It checks out
  `github.sha` (the synthetic merge result for pull requests), installs the locked Node/npm toolchain from `.nvmrc` plus
  `npm run setup`, then runs `npm run check:all` with read-only `contents: read` permissions, one pinned full-SHA action
  per step, concurrency cancellation keyed on `github.workflow`/`github.ref`, and a 30-minute timeout. It publishes,
  tags, or uploads nothing; `.github/workflows/publish-release.yml` remains the sole tag-triggered publisher and still
  reruns `check:all` itself before publishing.
- `.github/CODEOWNERS` requires review from the quality-system owner for workflows, hooks, all root quality/toolchain
  configuration, all `tools/`, schemas/types, quality contracts/tool tests/helpers/setup/scaling tests, and canonical
  quality/contributor documentation, maintained agent skills, and active execution plans. The contract test enumerates
  these required surfaces and fails if one is removed. CODEOWNERS review only takes effect once the following is
  configured as a GitHub branch ruleset for `main` (an owner action outside this repository's files):
  - require the `quality` check from `.github/workflows/quality.yml` to pass before merge;
  - require at least one CODEOWNERS review approval for pull requests that touch a path listed in `.github/CODEOWNERS`;
  - dismiss stale approvals whenever a quality-policy path changes after approval;
  - prohibit direct pushes to `main` (require pull requests for everyone, including administrators).
